Extract Model: Unified 3-Field Schema

extract[] Every entry has exactly 3 fields:
from 从哪里提取 cookies | localStorage | eval
name 存储为什么名字 credential store key
key 取什么 * | name | glob | dot-path
required[] Provider-level completion check. Format: name.field session.reddit_session — cookie "reddit_session" must exist in extract "session" xoxc-token — extract "xoxc-token" must be non-empty (no dot = whole value check)
Provider-level fields
sourcebrowser | prompt | env ttlwhen to re-extract ("7d", "12h") requiredcompletion criteria (name.field format) cookiePathsextra URL paths to query cookies from applyhow to inject into HTTP requests networkProxySOCKS proxy for browser

Design Decisions (Locked In)

1. source = strategy — browser | prompt | env. Determines HOW to acquire.
2. extract[] is uniform — 3 fields only: from, name, key. Same schema for cookies/localStorage/eval.
3. required = completion check — name.field format. Poll until all satisfied.
4. OAuth2 → browser + localStorage — No separate oauth2 strategy. MSAL tokens extracted via localStorage key pattern.
5. CDP-first extraction — All browser sub-extractors (cookies, localStorage, eval) use CDP. No Playwright for extraction.
6. extract + apply always required — No implicit defaults. Explicit is better than implicit. Every provider declares both.
7. Flat config — No strategy: / strategyConfig: / config: nesting. Everything at provider level.
