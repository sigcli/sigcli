MCP auth servers solve a real problem — but they lock you into one protocol, one ecosystem, and one vendor's systems. Signet separates auth from everything else, so it works with any agent, any system, any transport.

The answer

What is Signet?

A central authentication CLI that bridges browser-based SSO to headless, programmatic access.

Human logs in once via browser. Signet captures and stores the credentials. Any agent, script, or CLI can retrieve them instantly — locally or on remote machines.

In action

From zero to authenticated

signet
# Human: one-time setup
$ sig init
Config created

# Human: authenticate via browser
$ sig login https://jira.corp.com
Opening browser...
Authenticated as you@corp.com

# Agent: get credentials instantly
$ sig get jira
{"Cookie": "JSESSIONID=..."}

# Agent: make authenticated request
$ sig request https://jira.corp.com/api/issue/KEY-1
200 OK

sig init

Auto-detects browser, creates config. Once per machine.

sig login <url>

Human authenticates via real browser SSO. Signet captures cookies, tokens, headers.

sig get <provider>

Agent retrieves credentials as JSON headers. Zero SDK needed — just shell out.

sig sync push <remote>

Push credentials to CI, containers, remote servers over SSH.

sig watch start

Background daemon auto-refreshes before expiry. Agents never break.
